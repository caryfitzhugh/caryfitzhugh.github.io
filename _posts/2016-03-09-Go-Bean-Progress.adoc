# Go-Bean Progress!	

  go-bean install
  
Now you have access to go-bean.

Currently all you can do is go-bean config and init.

Init will let you set values in the .go-bean file.  
Config will print out those values.

The only value currently needed is the ApplicationName.

Eventually we will add environments (so we can alias things) and have defaults.
But that will come soon.

Now I am working on releasing snapshots.

It made me realize I want a 

 go-bean snapshot

and

 go-bean release
 
so that we can keep these two things separate.

Both will package up the code into a docker , push it to the ECR, and create an application version, and then deploy that.  But release will do it for "real" builds.
